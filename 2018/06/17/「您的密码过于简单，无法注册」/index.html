「您的密码过于简单，无法注册」

今日热点 知乎日报 2018/06/17

我认为检测密码强度（通用性检测和关联性检测除外）其实是程序员的一种怨念，以及对密码安全策略的错误理解。
所有银行的银行卡密码都是 6 位纯数字，延续了 20 多年依然没出现大问题。在良好的密码安全策略实施情况下，高强度密码其实对终端用户安全提升并不显着；反之网站是不是容易被入侵、密码传输是否采用安全套接层、密码碰撞是否可以即时发现并予与拦截、密码存储是否采取一码一 Salt 的 hash 策略等对密码安全的影响要更大。
且不说二流网站与否，目前几乎绝大部分网站都会做密码脆弱性检测。其中包含密码长度、密码所含字符成分等。
还有个别网站还会判断密码与用户名关系以及密码的通用性等。譬如用户名为“zhangsan” ，密码为“zhangsan”或“zhangsan19900516”，或者密码为“123456”或“111111”，这种判断是很有意义的，原因也显而易见——社工。服务端也无法有很好的策略防范。 不过大部分网站都没这个功能。
但去判断密码长短、包含字符内容其实是体现了系统在安全防护策略上执行不彻底，以及在系统被攻破情况下没有信心。因为这种直观的限定主要是为了防御两种情况：第一种就是暴力登录破解；第二种是脱裤后的 Hash 彩虹表碰撞。
第一种暴力登录破解情况，只要采取数次密码输入错误后进行手机验证码、邮箱验证码，或类似 google authenticator 等双重验证，以及登录过程中做 IP 区域限定及锁定策略就可以了。
而第二种彩虹表碰撞情况，可采取多次多种 hash 算法混合 + 用户唯一高复杂度 Salt 方式进行规避。这样彩虹表也将一无是处。虽然如果通过暴力破解方式依然是较短的纯数字或字符的密码更容易爆破，但与其都考虑到了这个层面，不如更多从系统安全、数据安全上多下功夫。譬如 passport 服务器单独放在一台服务器上，对业务机提供服务 api，在 passport 机上去验证中间 hash 给予反馈，而不是业务机直连 passport 机查询数据库，来尽量达到避免 passport 机因为业务机被端而被直接脱裤的厄运。
当然即便在这些安全防护策略都实施情况下，依旧无法避免被脱裤，无法避免密码碰撞。但同样，这种情况下，密码的长短和复杂度也起不了大作用。
所以总而言之，对于二流网站来讲，因为密码安全策略实施不到位，所以反而验证密码强度有一定作用。而对于一流网站来讲，程序员和安全策略实施者就要充分考虑下你们为什么还要验证个密码复杂度了。是为了给用户一种我很安全的感觉？还是为了寻求在密码安全防护上的心理安慰？
当然最后还是要鄙视一下某些“一流”的网站，限定密码最大长度不超过 8 位，密码中还不能包含特殊字符到底是为个啥？
问题：a. 我觉得很奇怪，总有人拿银行卡的 6 位数密码和网站的密码来比。早些年的银行卡就算知道密码，拿不到卡片也是没任何用的啊。再说网站，任何一个链接过去就可以让全世界所有对这个网站感兴趣的黑客开始琢磨研究，这两者的难易程度有的比吗？
b. 问题是早些年只有拿着卡去取款机才能取款啊，一般的犯罪分子光知道密码几乎没用。
回复：a. 有的比，所以文中提到 IP 区域限定和双重验证。银行卡也是这几种策略罢了。现在做好密码安全策略并不是达不到银行的水平，只是才刚刚发展起来。目前市场上已经有了一些很牛掰的产品，譬如 RSA token、google authenticator、360ID 动态口令。手机验证码验证、指纹瞳孔和面部识别等也能达到很好的效果，只不过真正发展起来还需要些时间。未来一定是一个弱化密码或者无密码的社会，依靠密码来提高安全性是个阶段性的解决方案，不会是未来的主流。
补充摘用：最新上线的桌面端 Chrome 67 版本中允许大多数网站实现免密码登陆方式，这就意味着用户不再需要通过密码管理器来搜索某个特定的登陆凭证。免密登陆只要依托于 Web Authentication 标准，由 FIDO 联盟和 W3C 于今年 3 月份发布，允许用户通过指纹读卡器、YubiKeys 等 USB 密钥等方式快速登陆在线服务。W3C 推出的 WebAuthn API 能为每个站点启用更强大的、唯一的、基于公钥的凭证，从而减少了因某个站点密码被盗导致其它站点易受影响的风险。WebAuthn 可集成到浏览器和网络平台基础设施中，为用户提供新方法在 web、浏览器、不同站点和设备上进行安全认证（https://www.cnbeta.com/articles/soft/731881.htm）。
b. 这正是银行的一些安全特性。但互联网不是达不到，只是网络验证机制发展落后罢了。captcha 验证机制出现不到 20 年，手机号验证出现不到 10 年，身份证活体验证出现不到 5 年。网络是需要逐步完善和进步的，这不能表明复杂密码的必要性。这些年你应该已经看到验证不单单靠密码了吧。
